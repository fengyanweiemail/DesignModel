##### 1、桥接模式
```text
    将抽象与实现分离，使他们可以独立变化。用组合关系代替继承关系来实现，从而降低了抽象和实现
这两个可变维度的耦合性，两个层次可以独立变化。

    解决类爆炸问题，替代了多层继承的实现方式，减少子类个数，降低了维护成本，基于类的最小设计原则，
    
    使用需要识别出业务中两个独立变化的过程，因此有一定的局限性；
    
    扩展的话比较简单、明了；
```
##### 2、角色说明
```text
    调用者：桥接模式的客户端；
    抽象类（Abstraction）：聚合接口类，实际上就是具体的接口实现类，充当桥接类的角色，把接口实现
和抽象定义桥接起来了；
    具体抽象类：是桥接类的子类；
    接口：抽象者行为实现类的接口；可以理解成抽象对象行为的实现，比如：直板手机怎么打开，触屏手机怎么
打开；又比如:即时消息，延迟消息，手机qq怎么发送，短信怎么发送、邮件怎么发送等等；银行转账：柜台转账，ATM转账，普通用户怎么转，会员怎么转等；
具体的抽象要根据实际业务进行分析提取；
    具体接口实现类：行为的接口类的实现；
```
