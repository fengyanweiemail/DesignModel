##### 1、享元设计模式
```text
    利用共享技术来有效的支持大量细微粒度的复用，它通过共享存在的对象来大幅度减少需要创建对象的数量
、避免大量相似类的开销，从而提高系统资源的利用率。

    必须抽象出内部状态（共享的部分）和外部状态（不能共享的部分），增加了系统的复杂性。
        内部状态：不会随环境的变化而变化；
        外部状态：是享元部分不能共享的，是随环境实时变化的；
    读取享元部分的外部状态会使得运行时间稍微增长。
    
    
```
##### 2、享元模式的角色说明
```text
    抽象享元角色（flyWeight）：所有具体享元角色的基类，为具体的享元规范需要实现的接口，非享元的外部
状态以方法参数的形式传入。
    具体享元角色（concrete flyWeight）:实现抽象享元角色中封装的角色。
    非享元角色（unshareble flyWeight）:是不可以共享的外部状态，它以参数的形式传入具体的享元方法中。
    享元工厂角色（flyWeight factory）：负责创建和管理享元角色，当客户对象访问一个享元对象时，工厂
里面判断是否存在，存在直接返回，不存在，则创建一个新对象。（里面应该存在一个集合，可以是内存等。）

```
