##### 1、建造者模式（生成器模式--用来产出复杂的对象）
```text
    一般适用于复杂对象的创建，建造者用来进行封装，使用建造者模式可以使客户端不必知道
产品内部组成的细节;
    另：建造者模式所创建的产品一般具有较多的共同点，如果产品之间差异性很大，则不适合
建造者模式；
    如果产品内部变化复杂，考虑建造者模式；
```
##### 2、抽象工厂和建造者
```text
    抽象工厂是实现产品家族创建，一个产品家族是这样一系列的产品；具有不同分类维度的产品组合，
工厂模式不需要关心构建过程，只关心什么产品由什么工厂生产即可。而建造者模式则是要求按照
指定的蓝图建造产品，它的主要目的是通过组装零配件而产生一个新产品

```
##### 3、建造者模式的角色分类
```text
    product(产品角色):一个具体的产品对象，包含多个组成部件的复杂对象，由具体的建造者创建各个
部件。就是说建造者建造出来的具体对象
    builder(建造者)：包含创建一个product产品对象各个部件的接口方法，通常还包含一个返回
复杂产品方法的getResult();就是创建对象需要的所有部件，那么创建部件需要方法啊，所以就有了
建造者。
    concreteBuilder(具体建造者)：实现builder接口，完成创建复杂对象部件的各个方法实现。
    director（指挥者）:它调用建造者对象中的部件构造和组装方法，完成复杂对象的创建，在指
挥者中不涉及具体产品的信息。1、隔离了客户与对象生产的过程；2、负责控制产品对象生产过程。

```
