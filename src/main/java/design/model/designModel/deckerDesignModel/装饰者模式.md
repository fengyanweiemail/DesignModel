##### 1、装饰者模式
```text
    指在不改变现有结构的情况下，动态地给该对象增加一些职责（既增加其额外功能）的模式，它属于对象
结构型模式；

    解决类爆炸问题，装饰比继承更加灵活；
    
    创建多个不同的装饰类，实现多个不同的组合；
    
    过度使用装饰模式，会创建很多类，使程序变得复杂；

```
##### 2、角色说明
```text
    抽象构件：定义一个抽象接口以规范准备接收附加责任的对象；
    具体构件：实现抽象构件，通过装饰角色为期添加一些职责（可以抽象出一个缓冲层，用来提取具体构件的共同点）；
    抽象装饰：继承抽象构件，并包含‘具体构件’的实例（也就是将父类聚合到自己本身），可以通过其子类（具体装饰）扩展 
具体构件的功能；
    具体装饰：实现抽象装饰的具体方法，并给具体构件对象添加附加的责任；
```
